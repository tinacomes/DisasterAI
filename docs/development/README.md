# Development History (internal)

⚠️ **Remove this directory before submitting the repository externally**
(e.g. before archiving for double-anonymous review or attaching to the
submission): these are internal working documents, and the review/fix logs
reference development details that are not part of the published methods.
Deleting the directory does not affect the model, experiments, or workflows —
nothing in the pipeline reads these files. Note that the files remain in git
history; for a fully clean export, archive a release tarball (Zenodo does
this automatically) rather than the git history itself.
